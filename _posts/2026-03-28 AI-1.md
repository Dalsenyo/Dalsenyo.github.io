## 들어가며

  

요즘 나는 Gemini로 캐릭터 컨셉 이미지를 만들고, 그걸 Meshy에 던져서 3D 모델을 뽑아내는 워크플로우를 자주 쓰고 있다. 자동 텍스처 입히고, 간단한 리깅 정도까지 해봤는데… 솔직히 결과물이 "와 이거 게임에 넣을 수 있겠다" 수준은 아니었다.

  

그래서 같은 도구를 쓰더라도 퀄리티를 더 끌어올릴 수 있는 방법이 없을까 고민했고, 최신 업데이트와 실무자들의 팁을 찾아보면서 꽤 유의미한 차이를 만드는 포인트들을 발견했다. 이 글에서는 Meshy를 중심으로, **같은 도구에서 결과물의 퀄리티를 확 올리는 실전 팁**을 정리한다.

  

---

  

## Meshy 6 — 뭐가 달라졌나

  

혹시 아직 Meshy 4나 5를 쓰고 있다면 Meshy 6으로 바꾸는 것만으로도 체감이 크다.

  

2025년 10월에 공개된 Meshy 6 Preview는 "조각 수준(sculpting-level)"의 디테일을 표방하면서 나왔다. 실제로 써보면 캐릭터나 유기체 모델에서 가장 차이가 큰데, 비례가 더 사실적이고 해부학적으로 일관된 형태가 나온다. 하드서페이스(기계나 차량 같은 딱딱한 표면)도 엣지가 더 선명하고 구조가 깨끗해졌다.

  

특히 캐릭터 모델링을 하는 입장에서는 **손가락이 뭉개지는 빈도**가 눈에 띄게 줄었다. 이전 버전에서는 손이 거의 장갑 수준이었는데, 6에서는 관절 구분이 어느 정도 살아있다. 물론 완벽하진 않지만, 후처리 작업량이 확실히 줄어든다.

  

> 💡 **참고:** Meshy 6로 생성한 모델은 무료 플랜에서는 다운로드할 수 없다. Pro 이상의 유료 플랜이 필요하다.

## 퀄리티를 올리는 가장 큰 변수: 멀티뷰 입력

  

이게 이 글에서 **가장 중요한 내용**이다.

  

지금 이미지 한 장만 던지고 있다면, 그건 AI에게 "앞면만 보고 뒷면도 알아서 추측해줘"라고 하는 것과 같다. 당연히 뒷면이 이상하게 나온다. 머리카락 뒤쪽이 납작하거나, 뒷주머니가 사라지거나, 뒤통수에 얼굴이 하나 더 생기는 현상을 경험해본 적 있을 것이다.

  

Meshy 6에서는 **멀티뷰(Multi-View)** 기능을 사용해서 메인 이미지 외에 최대 3장의 추가 이미지를 다른 각도에서 넣을 수 있다. 이러면 AI가 뒷면이나 옆면을 "추측"하는 게 아니라, 실제 레퍼런스를 보고 만들기 때문에 정확도가 훨씬 올라간다.

  

### 실전 워크플로우

  

```

1. Gemini에서 정면 이미지 생성

   → "character front view, A-pose, white background, full body"

  

2. 같은 캐릭터의 측면/후면도 생성

   → "character side view, same design as [설명], white background"

   → "character back view, same design as [설명], white background"

  

3. Meshy에서 Image to 3D 선택

  

4. 메인 이미지 업로드 후 Multi-view 토글 ON

  

5. 추가 이미지 3장 업로드

  

6. Generate

```

  

### 주의할 점

  

- **이미지 순서는 결과에 영향을 주지 않는다.** 그러니까 정면을 먼저 넣든 후면을 먼저 넣든 상관없다.

- **한 이미지에 여러 오브젝트나 혼합된 뷰를 넣지 마라.** 예를 들어 한 장에 정면+후면을 나란히 넣은 턴어라운드 시트는 오히려 AI를 혼란시킬 수 있다. 각 각도가 독립된 이미지여야 한다.

- **멀티뷰는 Meshy 6에서만 지원된다.** Meshy 5에서는 사용할 수 없다.

- **Pro/Studio 플랜에서만 사용 가능하다.**

  

### Gemini에서 멀티뷰용 이미지 잘 뽑는 팁

  

솔직히 Gemini에서 "같은 캐릭터의 다른 각도"를 일관되게 뽑는 건 쉽지 않다. 각도를 바꿀 때마다 디자인이 미묘하게 달라지기 때문이다.

  

이걸 최소화하는 방법:

  

1. **첫 이미지를 최대한 구체적으로 프롬프트한다.** 색상, 옷 디테일, 헤어스타일을 상세하게.

2. **다른 각도를 생성할 때 첫 이미지의 설명을 그대로 복붙한다.** "same character as described above, but from side view" 식으로.

3. **완벽한 일관성은 포기한다.** 어차피 Meshy가 멀티뷰를 종합해서 모델을 만들기 때문에, 대략적인 실루엣과 색감이 맞으면 충분하다. 디테일의 미세한 차이는 AI가 알아서 평균을 낸다.

4. **배경은 반드시 단색(흰색 추천)으로.** 복잡한 배경은 AI가 배경까지 3D로 만들려고 시도해서 모델이 지저분해진다.

## 입력 이미지 품질 극대화

  

멀티뷰를 안 쓰더라도, 입력 이미지 하나의 품질만 올려도 결과가 확 달라진다.

  

### 좋은 입력 이미지의 조건

  

| 항목 | 좋음 ✅ | 나쁨 ❌ |

|------|---------|---------|

| 배경 | 단색, 투명, 깨끗 | 복잡한 풍경, 다른 캐릭터 포함 |

| 조명 | 균일한 스튜디오 라이팅 | 강한 그림자, 역광 |

| 구도 | 피사체가 가운데, 전신 보임 | 잘린 부분 있음, 너무 작음 |

| 해상도 | 고해상도 (1024px 이상) | 흐리거나 저해상도 |

| 포즈 | A-Pose (팔 살짝 내림) | 복잡한 액션 포즈 |

  

### Image Enhancement 기능

  

Meshy에는 **Image Enhancement**라는 옵션이 있다. 켜면 업로드한 이미지를 AI가 자동으로 최적화해서 더 나은 모델 생성 결과를 만들어준다.

  

근데 항상 켜는 게 좋은 건 아니다.

  

- **켜면 좋은 경우:** 이미지가 약간 흐리거나, 배경이 완전히 깨끗하지 않을 때. AI가 알아서 정리해준다.

- **끄면 좋은 경우:** 이미지의 스타일을 정확하게 유지하고 싶을 때. Enhancement가 원래 의도한 색감이나 스타일을 바꿔버릴 수 있다. 특히 **툰/애니메이션 스타일**이라면 끄는 게 나을 때가 많다.

  

### A-Pose를 추천하는 이유

  

캐릭터를 생성할 때, Gemini 프롬프트에 "A-Pose"를 꼭 넣어라. A-Pose는 팔을 몸에서 약 45도 정도 벌린 자세를 말한다.

  

왜 이게 중요하냐면:

  

1. **리깅(뼈대 넣기)을 나중에 할 때 훨씬 수월하다.** 팔이 붙어있으면 겨드랑이 부분의 메시가 뭉개져서 리깅이 어렵다.

2. **Meshy의 자동 리깅도 A-Pose에서 가장 잘 작동한다.** Meshy에서 포즈 모드를 A-Pose나 T-Pose로 설정할 수 있는데, 입력 이미지 자체가 이 포즈면 결과가 더 정확하다.

3. **T-Pose(팔을 옆으로 쭉 편 자세)보다 A-Pose가 어깨 변형이 자연스럽다.** 요즘은 A-Pose가 업계 표준에 더 가깝다.

## 생성 후 퀄리티 올리기

  

모델이 생성되었다고 끝이 아니다. Meshy 안에서 할 수 있는 후처리가 몇 가지 있다.

  

### 리메시(Remesh)를 먼저, 텍스처는 나중에

  

이 순서가 중요하다. **텍스처를 입히기 전에 모델을 먼저 리메시하는 걸 추천한다.** 리메시를 하면 폴리곤 수가 줄어들고 더 깨끗한 텍스처 결과물이 나온다.

  

리메시란 메시(3D 모델의 그물망 구조)를 다시 정리하는 과정이다. AI가 만든 모델은 보통 삼각형이 마구잡이로 뒤엉켜 있는데 — 업계에서는 이걸 "triangle soup(삼각형 수프)"라고 부른다 — 리메시를 하면 이걸 더 정돈된 구조로 바꿔준다.

  

Meshy에서 슬라이더를 드래그하거나 숫자를 입력해서 원하는 폴리카운트 비율을 조절할 수 있다. 예를 들어 원본이 10,778 면(face)인데 4,000으로 줄이고 싶으면 비율을 0.43으로 설정하면 된다.

  

### Smart Healing으로 텍스처 결함 수정

  

AI 모델에서 자주 발생하는 문제: 뒷면에 얼굴이 하나 더 생기거나, 텍스처 이음새가 보이거나, 디테일이 이상하게 나온 경우.

  

Smart Healing 기능은 이런 작은 텍스처 결함을 콕 찍어서 수정할 수 있다.

  

```

사용법:

1. 모델을 적절한 각도로 조절

2. 오른쪽 툴바에서 "Smart Healing" 클릭

3. 올가미(lasso) 또는 브러시 도구로 수정할 텍스처 영역 선택

4. 필요하면 지우개로 모델 범위 밖의 영역을 제거

5. "Smart Healing" 클릭하여 선택 영역 수복

6. 만족할 때까지 "Redo" / "Undo" 사용

```

  

> ⚠️ **팁:** 편집 영역을 선택할 때 검은 배경 영역을 포함하지 않도록 주의하라. 모델 범위 밖의 영역이 포함되면 비정상적인 수리 결과가 나올 수 있다.

  

### AI 텍스처링을 반복적으로 활용하기

  

Meshy에서 자동 텍스처 한 번만 돌리고 끝내지 마라. **같은 메시에 텍스처를 무제한으로 만들 수 있고**, 프롬프트를 바꾸면서 색상, 패턴, 재질을 다르게 할 수 있다.

  

내 졸업작품이 툰/애니메이션 스타일이기 때문에, 텍스처 프롬프트에 스타일 키워드를 넣어봤다:

  

```

"cel-shaded, anime style, flat color with bold outlines"

"hand-painted stylized texture, vibrant colors"

"Japanese anime character texture, soft shading"

```

  

마음에 드는 게 나올 때까지 여러 번 돌리는 게 좋다. 크레딧을 아끼려고 한 번에 결정하려 하지 말고, 여러 변형을 만들어보고 비교하자.

## Meshy만이 답은 아니다

  

같은 컨셉 이미지를 여러 AI 3D 도구에 넣어보고 가장 나은 결과를 택하는 것도 좋은 전략이다.

  

### 비교할 만한 도구들

  

| 도구 | 특징 | 언제 써볼 만한가 |

|------|------|-----------------|

| **Tripo AI** | 생성 + 세그멘테이션 + 리토폴로지 + 리깅을 하나의 플랫폼에서 | 후처리까지 한 곳에서 끝내고 싶을 때 |

| **Autodesk Flow Studio (Wonder 3D)** | 2026년 3월 출시. 텍스트/이미지 to 3D + 리메시 + 텍스처 | Autodesk 생태계를 이미 쓰고 있을 때 |

| **3D AI Studio** | 여러 AI 모델을 한 곳에서 비교 가능 | 같은 이미지로 여러 모델을 비교해보고 싶을 때 |

| **Rodin (Hyper3D)** | 포토리얼리스틱 품질에 특화 | 사실적인 에셋이 필요할 때 |

  

도구를 하나만 고집하기보다, **상황에 따라 가장 좋은 결과물을 주는 도구를 유연하게 골라 쓰는 게 AI 시대의 스마트한 워크플로우**라고 생각한다.

## 전체 추천 워크플로우 요약

  

```

┌─────────────────────────────────────────────────────┐

│  1. Gemini에서 캐릭터 컨셉 이미지 생성               │

│     (단색 배경, 여러 각도, A-Pose)                    │

└──────────────────────┬──────────────────────────────┘

                       ▼

┌─────────────────────────────────────────────────────┐

│  2. Meshy 6 → Image to 3D (멀티뷰 ON, 포즈 설정)    │

└──────────────────────┬──────────────────────────────┘

                       ▼

┌─────────────────────────────────────────────────────┐

│  3. 리메시로 메시 정리                                │

│     → Smart Healing으로 텍스처 결함 수정              │

└──────────────────────┬──────────────────────────────┘

                       ▼

┌─────────────────────────────────────────────────────┐

│  4. 스타일에 맞는 AI 텍스처 여러 번 시도              │

└──────────────────────┬──────────────────────────────┘

                       ▼

┌─────────────────────────────────────────────────────┐

│  5. FBX로 내보내서 3ds Max에서 리토폴로지 + UV 정리   │

│     (→ 3편에서 자세히 다룸)                           │

└──────────────────────┬──────────────────────────────┘

                       ▼

┌─────────────────────────────────────────────────────┐

│  6. 언리얼 엔진으로 가져와서 커스텀 툰 셰이더 적용    │

│     (→ 5편에서 자세히 다룸)                           │

└─────────────────────────────────────────────────────┘

```

## 마치며

  

Meshy는 "버튼 하나 누르면 게임 에셋이 뚝딱 나오는 마법 도구"가 아니다. 하지만 **어떻게 쓰느냐에 따라 모델링 시간을 크게 단축할 수 있는 강력한 출발점**이 된다.

  

이번 글에서 다룬 핵심을 정리하면:

  

1. **Meshy 6을 써라** — 캐릭터 퀄리티가 눈에 띄게 올랐다

2. **멀티뷰를 활용해라** — 이미지 1장 vs 4장의 차이는 하늘과 땅이다

3. **입력 이미지에 신경 써라** — 쓰레기를 넣으면 쓰레기가 나온다 (GIGO)

4. **리메시 → 텍스처 순서를 지켜라** — 순서가 바뀌면 텍스처가 깨진다

5. **텍스처를 여러 번 돌려라** — 첫 시도가 최선이 아닐 확률이 높다

  

다음 글(2편)에서는 이런 AI 도구를 활용하면서도 포트폴리오에서 **나만의 가치**를 어떻게 보여줄 수 있는지를 다룬다. AI 시대에 게임 모델러로 취업하려면 어떤 전략이 필요한지, 업계의 솔직한 분위기와 함께.
